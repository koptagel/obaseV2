Kod Dosyalari

Serverin arka planinda calisan kodlar, kapsadigi ozelliklere ve islevlerine gore farkli dosyalarda bulunuyor. Her bir dosyanin icerisinde gerekli fonksiyonlar bulunuyor.

Asagidaki listede dosyalar ve hangi durumlarda bu dosyalardaki kodlarin kullanildigi yaziyor. Fonksiyon detaylari, dosya notebooklarinda bulunuyor.

  • TornadoServer
    • Server kodu bu dosyda bulunuyor. Servera gonderilen isteklerin gecerli olup olmadigi burada kontrol ediliyor. Ardindan, ilgili dosyalardaki kodlar calistiriliyor.
  • GenerateFilesForSql
    • Veritabani kurulumunu yapmak icin gereken text dosyalari bu dosyada olusturuluyor.
  • OfflineFunctions
    • Offline olarak (kurulum sirasinda veya veri tabani guncellemelerinde) calistirilmasi gereken kodlar bu dosyada bulunuyor. Satis tensorunun marginallerinin hesaplanmasi, musterilerin satis tahminlerinin yapilmasi gibi kodlari kapsiyor.
  • DatabaseInfoFunctions
    • Veritabani ile ilgileri bilgileri gosteriyor.
  • MappingFunctions
    • Musteri ve urunlerin gercek id degerleriyle, sistemde kullanilan index deger karsiliklarini hesaplayan fonksiyonlari bulunduruyor.
  • WeblogFunctions
    • Musterilerin web hareketleriyle alakali fonksiyonlar bu dosyada bulunuyor.
  • SalesFunctions
    • Urun satislariyla alakali fonksiyonlar, satis grafiklerinin cizimi gibi fonksiyonlar yer aliyor.
  • SalesMarginalFunctions
    • Urun satislariyla alakali fonksiyonlar, satis grafiklerinin cizimi gibi fonksiyonlar yer aliyor. Bu fonksiyonlar, onceden hazirlanmis marginal satis tensorlerini kullanarak hesaplama yapiyor.
  • RecommendationFunctions
    • Urun tavsiyeleriyle ilgili fonksiyonlar bu dosyada bulunuyor.
  • DistanceFunctions
    • Satis matrisleri arasindaki uzakligi hesaplamak icin gerekli olan fonksiyonlar bu dosyada yer aliyor.
  • NmfFunctions
    • NMF ile ilgili fonksiyonlari kapsiyor.
  • ProfileFunctions
    • Musteri profilleri ile ilgili islemler icin bu dosyadaki fonksiyonlar kullaniliyor.
  • SimilarityFunctions
    • Benzer musterileri bulmak ve bu musterilere urun onermek icin kullanilan kodlar bu dosyada yer aliyor.
  • HtmlPages
    • Eski api sayfasini olusturmak icin gerekli olan Html scripti bu dosyada olusturuluyor.

Offline Functions

This file contains operations which should be done offline. The functions in this file have main features; creating marginal tensors from sales and generating recommendations for customers.

This file also contains the codes required for setting up the server and updating the database.

Functions:

  • generateRecommendationsG3(db_name, criteria)
    • Given the criteria, this function generates recommendations for customers, based of item group 3 information.
  • updateSalesEstimationG3Table(db_name, criteria, salesMatrixEst)
    • Updates corresponding SalesEstimationG3 table in the database.
  • updateRecommendationsG3(db_name, criteria)
    • Combination of generateRecommendationsG3 and updateSalesEstimationG3Table functions.
  • generateRecommendations(db_name, criteria)
    • Given the criteria, this function generates recommendations for customers, based of item information.
  • updateSalesEstimationTable(db_name, criteria, salesMatrixEst)
    • Updates corresponding SalesEstimation table in the database.
  • updateRecommendations(db_name, criteria)
    • Combination of generateRecommendations and updateSalesEstimationTable functions.
  • updateSalesEstimationTableWithThreshold(db_name, criteria, salesMatrixEst, threshold)
    • Given a threshold value, this function updates the SalesEstimation table by removing values less than the threshold.
  • updateRecommendationsWithThreshold(db_name, criteria, threshold)
    • Combination of generateRecommendations and updateSalesEstimationTableWithThreshold functions.
  • updateMarginalSalesTensorMat(db_name, desiredFields, desiredShapes, numCustomers)
    • By using SalesTensor table in the database, this function generates the marginals of the tensor and stores them in .mat files for further use.
  • updateAllMarginalSalesTensorMat(db_name)
    • This function generates all required marginal tensors by using updateMarginalSalesTensorMat function.

There two additional methods which are not used in the latest implementation in the system:

  • updateMarginalSalesTensor(db_name, desiredFields)
    • This function calculates a marginal of the SalesTensor table and stores the result in a table in the database.
  • updateAllMarginalSalesTensorMat(db_name)
    • This function generates all required marginal tensors by using updateMarginalSalesTensor function.

There are also two Python files in this file:

  • OfflineFunctionsForSetUp.py
    • This file is required for setting up the database in the server for the first time.
    It contains functions to calculate recommendations for customers, generate SalesEstimation and SalesEstimationG3 tables in the database.
  • OfflineFunctionsForUpdate.py
    • This file is required for updating the marginal tensors and sales recommendations, if the sales records in the database are changed.

Functions:

  • getCustomerIndex(db_name, customerId)
    • Given the CustomerId, returns CustomerIndex value.
  • getCustomerId(db_name, customerIndex)
    • Given the CustomerIndex, returns CustomerId value.
  • getItemIndexWithItemId(db_name, itemId)
    • Given the ItemId, returns ItemIndex value.
  • getItemIdWithItemIndex(db_name, itemIndex)
    • Given the ItemIndex, returns ItemId value.
  • getItemG3IndexWithItemId(db_name, itemId)
    • Given the ItemG3Index, returns ItemIndex value.
  • getItemG3IdWithItemIndex(db_name, itemIndex)
    • Given the ItemIndex, returns ItemG3Id value.
  • getItemG3IndexWithItemIndex(db_name, itemIndex)
    • Given the ItemIndex, returns ItemG3Index value.
  • getItemG3IdWithItemG3Index(db_name, itemG3Index)
    • Given the ItemG3Index, returns ItemG3Id value.

Sales Functions

This notebook contains functions which fetch the sales data from database and display the plots. There are also functions for getting sales estimations from database.

Functions:

  • getSalesMatrixOfCustomer(db_name, customerIndex, desiredFields, plotCriteria, shapes)
    • Given the desired axes, this function returns the sales matrix of the customer.
  • getSalesSlotMatrixOfCustomer(db_name, customerIndex, desiredFields, plotCriteria, shapes, TimePoints, TimePointsY)
    • Given the desired axes and time slot properties, this function returns sales matrix of the customer.
  • getSalesHistogramOfCustomer(db_name, customerIndex, desiredField, plotCriteria, shape1)
    • Given the desired axis, this function returns the sale histogram of the customer.
  • getCustomerSales(DB_NAME, customerIndex, criteria, ax1, ax2, TimePoints, TimePointsY
    • This function is the combination of getSalesMatrixOfCustomer, getSalesSlotMatrixOfCustomer and getSalesHistogramOfCustomer functions.
  • getCustomerSalesMap(DB_NAME, customerIndex, plotTitle, criteria, ax1, ax2, TimePoints, TimePointsY)
    • This function plots the sales of the customer according to given parameters.
  • getSalesMatrix(db_name, criteria, shapes)
    • This function returns marginal of sales tensor in the form of (Customers vs ItemGroup3s).
  • getSalesMatrixWithoutG3(db_name, criteria, shapes)
    • This function returns marginal of sales tensor in the form of (Customers vs Items).
  • getSalesEstimationG3OfCustomer(db_name, customerIndex, criteria, shape1)
    • This function returns sales estimation of the customer based on item group 3s.
  • getSalesEstimationOfCustomer(db_name, customerIndex, criteria, shape1)
    • This function returns sales estimation of the customer based on items.

Sales Marginal Functions

This notebook contains functions related to marginal sales tensor tables in the database and .mat files.

Functions:

  • getSalesMatrixOfCustomerFromMarginalMats(dataDict, customerIndex, plotCriteria)
    • Given the desired axes, this function returns the sales matrix of the customer using corresponding marginal tensor with .mat file extension.
  • getSalesHistogramOfCustomerFromMarginalMats(dataDict, customerIndex, ax1, plotCriteria)
    • Given the desired axiss, this function returns the sales histogram of the customer using corresponding marginal tensor with .mat file extension.
  • getSalesSlotMatrixOfCustomerFromMarginalMats(dataDict, customerIndex, ax1, ax2, plotCriteria, TimePoints, TimePointsY)
    • Given the desired axes and time slot information, this function returns the sales matrix of the customer using corresponding marginal tensor with .mat file extension.
  • getCustomerSalesFromMarginalMats(dataDict, customerIndex, criteria, ax1,ax2, TimePoints, TimePointsY)
    • This function is the combination of getSalesMatrixOfCustomerFromMarginalMats, getSalesHistogramOfCustomerFromMarginalMats and getSalesSlotMatrixOfCustomerFromMarginalMats functions.

The functions below are not used in the current implementation of server. These methods get marginal sales tensor from database tables, not from .mat files.

  • getSalesMatrixOfCustomerFromMarginals(db_name, customerIndex, desiredFields, plotCriteria, shapes):
    • Given the desired axes, this function returns the sales matrix of the customer using corresponding marginal tensor table in the database.
  • getCustomerSalesFromMarginals(DB_NAME, customerIndex, criteria, ax1, ax2, TimePoints, TimePointsY)
    • Given the desired axes, this function returns the sales of the customer using corresponding marginal tensor table in the database.

Functions:

Similarity Functions

This notebook contains functions related to 'similar customers' feature of the system. Basically, the functions in this file find similar customers to a specific customer and generate product recommendations for these customers.

Functions:

  • getSimilarCustomers(db_name, customerIndex, criteria, ax1, ax2, TimePoints, TimePointsY, searchType, distanceType, numCustomers, minPercentage, profileId)
    • Given the parameters and the customer index, this function returns the customer lists (consist of id, percentage pairs) and the minimum & maximum distance values to the given customer.
  • getGroupRecommendations(db_name, baseCount, criteria, recommenderType,numRecItems, shape1, customersData)
    • Given the customer list and the choices, this function returns suggested product list.
  • getSimilarCustomersWithProducts(db_name, customerIndex, criteria, ax1, ax2, TimePoints, TimePointsY, searchType, distanceType, numCustomers, minPercentage, profileId, numRecItems, recommenderType, shape1, baseCount)
    • This function is the combination of getSimilarCustomers and getGroupRecommendations functions.
