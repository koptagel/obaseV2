Gerekli Kutuphaneler

Serverda calisan kodlar Python ile yazildi. Programin eksiksiz calisabilmesi su kutuphanelerin kurulmasi gerekiyor:

  • numpy
  • scipy
  • sklearn
  • tornado
  • matplotlib
  • requests
  • datetime
  • networkx
  • sqlite3

Kodlara Erisim

Server kodlari su anda Github'daki bir repository'de tutuluyor.

git clone https://github.com/koptagel/obaseV2.git komutuyla server kodlari indirilebilir.

Veritabani Kurulumu

Git'ten server kodlari indirildikten sonra, veritabanini olusturmak gerekiyor. Veritabanini olusturmak icin gerekli olan 3 dosya, txtFiles klasorunde bulunuyor. Sirasiyla asagidaki islemlerin yapilmasi gerekiyor:

  • Komut satirindan server kodlarinin oldugu klasore gelinir.
  • cd database
  • sqlite3 ObaseDb.db
  • .mode csv
  • .separator \t

  • Create Table CustomerMapping (CustomerIndex Int, CustomerId Int);
  • .import ../txtFiles/CustomerMapping.txt CustomerMapping

  • Create Table ItemMapping (ItemIndex Int, ItemId Int, ItemG3Index Int, ItemG3Id Int, ItemG3Ds Text, ItemDs Text);
  • .import ../txtFiles/ItemMapping.txt ItemMapping

  • Create Table SalesTensor (WeekIndex Int, DowIndex Int, HourIndex Int, ItemIndex Int, ItemG3Index Int, CustomerIndex Int, Amount Real);
  • .import ../txtFiles/SalesTensor.txt SalesTensor

  • Create Table ProfileMapping (ProfileId Int, ProfileDs Text);
  • Create Table ProfileCustomers (ProfileId Int, CustomerId Int, CustomerPercentage Int);
  • Create Table ProfileItems (ProfileId Int, ItemId Int, ItemPercentage Int);
  • .exit

Bu komutlarla veritabaninda ihtiyac duydugumuz CustomerMapping, ItemMapping, SalesTensor, ProfileMapping, ProfileCustomers ve ProfileItems tablolari olusturuluyor.

Satis Tahmini Tablolarinin Olusturulmasi

Tavsiye sistemi belirli bir tarih araligini kapsadigi icin, o tarihler arasindaki satis tahminlerini onceden hesaplayip veritabanina ekliyoruz.

Hem urun hem de urun grup 3 bazli urun tavsiye tablolarini olusturmak icin asagidaki islemleri yapmak gerekiyor:

  • Komut satirindan server kodlarinin oldugu klasore gelinir.
  • python3 OfflineFunctionsForSetUp.py

OfflineFunctionsForSetUp.py kodunu calistirdigimizda veri tabanina satis tahminleriyle alakali 4 yeni tablo ekleniyor ve kurulum tamamlaniyor.

Veritabani

Serveri Calistirma

Kurulum tamamlandiktan sonra kodlarin oldugu dizine ulasip asagidaki komutu yazmak gerekiyor:

nohup python3 TornadoServer.py &

Boylelikle server kodu baslatilacak ve komut satiri kapatilsa bile server arka planda calismaya devam edecek.

Sistemi Guncelleme

Serveri yeni satis verileriyle birlikte calistirilmak istenildiginde (CustomerMapping, ItemMapping veya SalesTensor tablosunda degisiklik olursa), tavsiye sisteminin yeniden calistirilip ilgili tablolarin guncellenmesi gerekiyor. Bu guncellemeleri yapmak icin:

  • ps -ef komutuyla hali hazirda calisan server kodunun (TornadoServer.py) PID degerini goruntuleyip,
  • kill PID komutuyla serveri durdurmak gerekiyor.
  • python3 OfflineFunctionsForUpdate.py komutuyla veritabaninda gerekli guncellemeler yapiliyor.

Ardindan server tekrar calistirildiginda, sistem guncel verilerle islem yapacak hale gelecek.
